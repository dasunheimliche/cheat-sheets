## A - Mix `Path`, `Query`, and Body Parameters

**Definición:**  
Puedes mezclar parámetros de ruta (`Path`), query parameters (`Query`) y parámetros del cuerpo de la solicitud (`Body`) en una misma función. FastAPI sabrá cómo manejarlos automáticamente.

**Ejemplo:**

```python
from typing import Annotated
from fastapi import FastAPI, Path
from pydantic import BaseModel

app = FastAPI()

class Item(BaseModel):
    name: str
    description: str | None = None
    price: float
    tax: float | None = None

@app.put("/items/{item_id}")
async def update_item(
    item_id: Annotated[int, Path(title="The ID of the item to get", ge=0, le=1000)],
    q: str | None = None,
    item: Item | None = None,
):
    results = {"item_id": item_id}
    if q:
        results.update({"q": q})
    if item:
        results.update({"item": item})
    return results
```

**Descripción del ejemplo:**

- `item_id` es un parámetro de ruta con validaciones (`ge=0`, `le=1000`).
- `q` es un query parameter opcional.
- `item` es un parámetro del cuerpo de la solicitud, opcional.

## B - Multiple Body Parameters

**Definición:**  
Puedes declarar múltiples parámetros del cuerpo de la solicitud. FastAPI esperará un JSON con claves que coincidan con los nombres de los parámetros.

**Ejemplo:**

```python
from fastapi import FastAPI
from pydantic import BaseModel

app = FastAPI()

class Item(BaseModel):
    name: str
    description: str | None = None
    price: float
    tax: float | None = None

class User(BaseModel):
    username: str
    full_name: str | None = None

@app.put("/items/{item_id}")
async def update_item(item_id: int, item: Item, user: User):
    results = {"item_id": item_id, "item": item, "user": user}
    return results
```

**Descripción del ejemplo:**

- FastAPI esperará un JSON con las claves `item` y `user`, cada una con sus respectivos datos.
- El cuerpo de la solicitud debe verse así:

  ```json
  {
    "item": {
      "name": "Foo",
      "description": "The pretender",
      "price": 42.0,
      "tax": 3.2
    },
    "user": {
      "username": "dave",
      "full_name": "Dave Grohl"
    }
  }
  ```

## C - Singular Values in Body

**Definición:**  
Puedes declarar valores individuales en el cuerpo de la solicitud usando `Body`. Esto es útil cuando necesitas un valor adicional que no forma parte de un modelo.

**Ejemplo:**

```python
from typing import Annotated
from fastapi import Body, FastAPI
from pydantic import BaseModel

app = FastAPI()

class Item(BaseModel):
    name: str
    description: str | None = None
    price: float
    tax: float | None = None

class User(BaseModel):
    username: str
    full_name: str | None = None

@app.put("/items/{item_id}")
async def update_item(
    item_id: int, item: Item, user: User, importance: Annotated[int, Body()]
):
    results = {"item_id": item_id, "item": item, "user": user, "importance": importance}
    return results
```

**Descripción del ejemplo:**

- `importance` es un valor individual en el cuerpo de la solicitud.
- FastAPI esperará un JSON con una clave adicional `importance`:

  ```json
  {
    "item": {
      "name": "Foo",
      "description": "The pretender",
      "price": 42.0,
      "tax": 3.2
    },
    "user": {
      "username": "dave",
      "full_name": "Dave Grohl"
    },
    "importance": 5
  }
  ```

## D - Multiple Body Params and Query

**Definición:**  
Puedes combinar múltiples parámetros del cuerpo de la solicitud con query parameters. FastAPI manejará ambos tipos de parámetros automáticamente.

**Ejemplo:**

```python
from typing import Annotated
from fastapi import Body, FastAPI
from pydantic import BaseModel

app = FastAPI()

class Item(BaseModel):
    name: str
    description: str | None = None
    price: float
    tax: float | None = None

class User(BaseModel):
    username: str
    full_name: str | None = None

@app.put("/items/{item_id}")
async def update_item(
    item_id: int,
    item: Item,
    user: User,
    importance: Annotated[int, Body(gt=0)],
    q: str | None = None,
):
    results = {"item_id": item_id, "item": item, "user": user, "importance": importance}
    if q:
        results.update({"q": q})
    return results
```

**Descripción del ejemplo:**

- `q` es un query parameter opcional.
- `importance` es un valor individual en el cuerpo de la solicitud con una validación (`gt=0`).

## E - Embed a Single Body Parameter

**Definición:**  
Puedes hacer que FastAPI espere un JSON con una clave que contenga el cuerpo de la solicitud, incluso si solo hay un parámetro del cuerpo.

**Ejemplo:**

```python
from typing import Annotated
from fastapi import Body, FastAPI
from pydantic import BaseModel

app = FastAPI()

class Item(BaseModel):
    name: str
    description: str | None = None
    price: float
    tax: float | None = None

@app.put("/items/{item_id}")
async def update_item(item_id: int, item: Annotated[Item, Body(embed=True)]):
    results = {"item_id": item_id, "item": item}
    return results
```

**Descripción del ejemplo:**

- FastAPI esperará un JSON con una clave `item` que contenga los datos del modelo:

  ```json
  {
    "item": {
      "name": "Foo",
      "description": "The pretender",
      "price": 42.0,
      "tax": 3.2
    }
  }
  ```

## F - Recap (Resumen)

**Definición:**  
Puedes agregar múltiples parámetros del cuerpo de la solicitud, valores individuales y query parameters en una misma función. FastAPI manejará la validación, documentación y extracción de datos automáticamente.

**Ejemplo:**

```python
from typing import Annotated
from fastapi import Body, FastAPI, Path
from pydantic import BaseModel

app = FastAPI()

class Item(BaseModel):
    name: str
    description: str | None = None
    price: float
    tax: float | None = None

@app.put("/items/{item_id}")
async def update_item(
    item_id: Annotated[int, Path(title="The ID of the item to get", ge=0, le=1000)],
    item: Item,
    importance: Annotated[int, Body(gt=0)],
    q: str | None = None,
):
    results = {"item_id": item_id, "item": item, "importance": importance}
    if q:
        results.update({"q": q})
    return results
```

**Descripción del ejemplo:**

- `item_id` es un parámetro de ruta con validaciones.
- `item` es un parámetro del cuerpo de la solicitud.
- `importance` es un valor individual en el cuerpo con una validación.
- `q` es un query parameter opcional.
