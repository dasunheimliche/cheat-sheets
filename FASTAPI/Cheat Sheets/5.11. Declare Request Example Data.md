## A - Extra JSON Schema data in Pydantic models (Datos adicionales en el esquema JSON de modelos Pydantic)

**Definición:**  
Puedes agregar ejemplos de datos a un modelo Pydantic que se incluirán en el esquema JSON generado. Esto es útil para documentar cómo deben ser los datos que tu aplicación puede recibir.

**Ejemplo:**

```python
from fastapi import FastAPI
from pydantic import BaseModel

app = FastAPI()

class Item(BaseModel):
    name: str
    description: str | None = None
    price: float
    tax: float | None = None

    model_config = {
        "json_schema_extra": {
            "examples": [
                {
                    "name": "Foo",
                    "description": "A very nice Item",
                    "price": 35.4,
                    "tax": 3.2,
                }
            ]
        }
    }

@app.put("/items/{item_id}")
async def update_item(item_id: int, item: Item):
    results = {"item_id": item_id, "item": item}
    return results
```

**Descripción:**  
En este ejemplo, se define un modelo `Item` con un ejemplo de datos que se incluirá en el esquema JSON generado. Esto ayuda a los usuarios de la API a entender cómo deben estructurar los datos.

## B - Field additional arguments (Argumentos adicionales en Field)

**Definición:**  
Puedes agregar ejemplos directamente en los campos de un modelo Pydantic usando `Field()`. Esto es útil para proporcionar ejemplos específicos para cada campo.

**Ejemplo:**

```python
from fastapi import FastAPI
from pydantic import BaseModel, Field

app = FastAPI()

class Item(BaseModel):
    name: str = Field(examples=["Foo"])
    description: str | None = Field(default=None, examples=["A very nice Item"])
    price: float = Field(examples=[35.4])
    tax: float | None = Field(default=None, examples=[3.2])

@app.put("/items/{item_id}")
async def update_item(item_id: int, item: Item):
    results = {"item_id": item_id, "item": item}
    return results
```

**Descripción:**  
Aquí, cada campo del modelo `Item` tiene un ejemplo específico definido con `Field(examples=[...])`. Esto proporciona ejemplos claros para cada campo individual.

## C - Examples in JSON Schema - OpenAPI (Ejemplos en el esquema JSON de OpenAPI)

**Definición:**  
Puedes declarar ejemplos en los esquemas JSON de OpenAPI para parámetros como `Body()`, `Query()`, `Header()`, etc. Esto es útil para mostrar ejemplos en la documentación de la API.

**Ejemplo:**

```python
from typing import Annotated
from fastapi import Body, FastAPI
from pydantic import BaseModel

app = FastAPI()

class Item(BaseModel):
    name: str
    description: str | None = None
    price: float
    tax: float | None = None

@app.put("/items/{item_id}")
async def update_item(
    item_id: int,
    item: Annotated[
        Item,
        Body(
            examples=[
                {
                    "name": "Foo",
                    "description": "A very nice Item",
                    "price": 35.4,
                    "tax": 3.2,
                }
            ],
        ),
    ],
):
    results = {"item_id": item_id, "item": item}
    return results
```

**Descripción:**  
En este ejemplo, se define un ejemplo de cuerpo de solicitud (`Body`) que se mostrará en la documentación de la API. Esto ayuda a los usuarios a entender cómo deben enviar los datos.

## D - Body with multiple examples (Cuerpo con múltiples ejemplos)

**Definición:**  
Puedes proporcionar múltiples ejemplos para un cuerpo de solicitud. Esto es útil para mostrar diferentes casos de uso en la documentación.

**Ejemplo:**

```python
from typing import Annotated
from fastapi import Body, FastAPI
from pydantic import BaseModel

app = FastAPI()

class Item(BaseModel):
    name: str
    description: str | None = None
    price: float
    tax: float | None = None

@app.put("/items/{item_id}")
async def update_item(
    item_id: int,
    item: Annotated[
        Item,
        Body(
            examples=[
                {
                    "name": "Foo",
                    "description": "A very nice Item",
                    "price": 35.4,
                    "tax": 3.2,
                },
                {
                    "name": "Bar",
                    "price": "35.4",
                },
                {
                    "name": "Baz",
                    "price": "thirty five point four",
                },
            ],
        ),
    ],
):
    results = {"item_id": item_id, "item": item}
    return results
```

**Descripción:**  
Aquí, se proporcionan tres ejemplos diferentes para el cuerpo de la solicitud. Esto permite a los usuarios ver diferentes formas de enviar datos.

## E - OpenAPI-specific examples (Ejemplos específicos de OpenAPI)

**Definición:**  
Puedes usar el parámetro `openapi_examples` para definir ejemplos específicos de OpenAPI. Esto es útil para agregar metadatos adicionales, como resúmenes y descripciones, a los ejemplos.

**Ejemplo:**

```python
from typing import Annotated
from fastapi import Body, FastAPI
from pydantic import BaseModel

app = FastAPI()

class Item(BaseModel):
    name: str
    description: str | None = None
    price: float
    tax: float | None = None

@app.put("/items/{item_id}")
async def update_item(
    item_id: int,
    item: Annotated[
        Item,
        Body(
            openapi_examples={
                "normal": {
                    "summary": "A normal example",
                    "description": "A **normal** item works correctly.",
                    "value": {
                        "name": "Foo",
                        "description": "A very nice Item",
                        "price": 35.4,
                        "tax": 3.2,
                    },
                },
                "converted": {
                    "summary": "An example with converted data",
                    "description": "FastAPI can convert price `strings` to actual `numbers` automatically",
                    "value": {
                        "name": "Bar",
                        "price": "35.4",
                    },
                },
                "invalid": {
                    "summary": "Invalid data is rejected with an error",
                    "value": {
                        "name": "Baz",
                        "price": "thirty five point four",
                    },
                },
            },
        ),
    ],
):
    results = {"item_id": item_id, "item": item}
    return results
```

**Descripción:**  
En este ejemplo, se definen tres ejemplos con metadatos adicionales, como resúmenes y descripciones. Esto enriquece la documentación de la API.

## F - Technical Details (Detalles técnicos)

**Definición:**  
Antes de OpenAPI 3.1.0, OpenAPI usaba una versión modificada de JSON Schema que no admitía el campo `examples`. FastAPI ahora usa OpenAPI 3.1.0, que admite `examples` en JSON Schema, lo que hace que la documentación sea más consistente.

**Ejemplo:**

```python
# En versiones anteriores de FastAPI, se usaba `example` en lugar de `examples`.
# Ahora, con OpenAPI 3.1.0, se prefiere `examples`.
```

**Descripción:**  
Este cambio permite una mejor integración con las herramientas de documentación, como Swagger UI, que ahora admite múltiples ejemplos en JSON Schema.
