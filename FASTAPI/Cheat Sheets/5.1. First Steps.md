## A - FastAPI

**Definición:**  
FastAPI es un framework moderno y rápido (de ahí su nombre) para construir APIs en Python. Es especialmente conocido por su velocidad y facilidad de uso, permitiendo crear APIs robustas con poco código. Además, genera automáticamente documentación interactiva para tu API.

**Ejemplo:**

```python
from fastapi import FastAPI

app = FastAPI()

@app.get("/")
async def root():
    return {"message": "Hello World"}
```

**Descripción del ejemplo:**  
Este código crea una API básica con FastAPI. La función `root()` se ejecuta cuando se accede a la ruta principal (`/`) usando el método HTTP GET. Devuelve un mensaje en formato JSON.

---

## B - Instancia de FastAPI

**Definición:**  
Una instancia de FastAPI es el objeto principal que maneja toda la configuración y operaciones de tu API. Es el punto de entrada para definir rutas, operaciones y otras configuraciones.

**Ejemplo:**

```python

from fastapi import FastAPI

app = FastAPI()
```

**Descripción del ejemplo:**  
Aquí, `app` es una instancia de la clase `FastAPI`. Este objeto se utiliza para definir rutas y operaciones en tu API.

---

## C - Path (Ruta)

**Definición:**  
En el contexto de una API, un "path" (o ruta) es la parte de la URL que sigue al dominio. Se utiliza para identificar recursos o acciones específicas en la API.

**Ejemplo:**

```python

@app.get("/items/")
async def read_items():
    return {"message": "Lista de ítems"}
```

**Descripción del ejemplo:**  
En este caso, `/items/` es el path. Cuando se accede a esta ruta usando GET, se ejecuta la función `read_items()` y se devuelve un mensaje en formato JSON.

---

## D - Operación HTTP

**Definición:**  
Una operación HTTP es un método que se utiliza para realizar acciones sobre un recurso en una API. Los métodos más comunes son GET, POST, PUT y DELETE.

**Ejemplo:**

```python

@app.post("/items/")
async def create_item():
    return {"message": "Ítem creado"}
```

**Descripción del ejemplo:**  
Aquí, `@app.post("/items/")` define una operación POST en la ruta `/items/`. Cuando se envía una solicitud POST a esta ruta, se ejecuta la función `create_item()`.

---

## E - Decorador

**Definición:**  
Un decorador en Python es una función que modifica el comportamiento de otra función. En FastAPI, los decoradores se utilizan para asociar funciones con rutas y operaciones HTTP.

**Ejemplo:**

```python

@app.get("/")
async def root():
    return {"message": "Hello World"}
```

**Descripción del ejemplo:**  
El decorador `@app.get("/")` asocia la función `root()` con la ruta `/` y la operación GET. Cuando se accede a esta ruta, se ejecuta la función.

---

## F - Función de Operación de Ruta

**Definición:**  
Es la función que se ejecuta cuando se realiza una solicitud a una ruta específica con una operación HTTP determinada. Esta función maneja la lógica de la solicitud y devuelve una respuesta.

**Ejemplo:**

```python

@app.get("/hello/")
async def say_hello():
    return {"message": "Hola, mundo!"}
```

**Descripción del ejemplo:**  
La función `say_hello()` es una función de operación de ruta. Se ejecuta cuando se accede a `/hello/` con el método GET y devuelve un mensaje en formato JSON.

---

## G - JSON Response

**Definición:**  
Es la respuesta que devuelve una API en formato JSON. FastAPI convierte automáticamente los objetos Python (como diccionarios o listas) en JSON.

**Ejemplo:**

```python

@app.get("/data/")
async def get_data():
    return {"name": "Alice", "age": 30}
```

**Descripción del ejemplo:**  
La función `get_data()` devuelve un diccionario que FastAPI convierte automáticamente en JSON: `{"name": "Alice", "age": 30}`.

---

## H - Documentación Interactiva

**Definición:**  
FastAPI genera automáticamente documentación interactiva para tu API usando Swagger UI y ReDoc. Esto permite probar y explorar la API directamente desde el navegador.

**Ejemplo:**

http://127.0.0.1:8000/docs
http://127.0.0.1:8000/redoc

**Descripción del ejemplo:**  
Al acceder a estas URLs, puedes ver y probar tu API sin necesidad de herramientas externas. Swagger UI y ReDoc son interfaces gráficas que muestran las rutas, operaciones y parámetros de tu API.

---

## I - OpenAPI

**Definición:**  
OpenAPI es un estándar para describir APIs. FastAPI genera automáticamente un esquema OpenAPI que describe todas las rutas, operaciones y parámetros de tu API.

**Ejemplo:**

http://127.0.0.1:8000/openapi.json

**Descripción del ejemplo:**  
Esta URL muestra el esquema OpenAPI en formato JSON. Este esquema es utilizado por herramientas como Swagger UI y ReDoc para generar la documentación interactiva.

---

## J - Uvicorn

**Definición:**  
Uvicorn es un servidor ASGI (Asynchronous Server Gateway Interface) que se utiliza para ejecutar aplicaciones FastAPI. Es rápido y eficiente para manejar solicitudes asíncronas.

**Ejemplo:**

```bash
fastapi dev main.py
```

**Descripción del ejemplo:**  
Este comando inicia un servidor Uvicorn para tu aplicación FastAPI. Escucha en `http://127.0.0.1:8000` y reinicia automáticamente cuando detecta cambios en el código.

---

## K - Esquema (Schema)

**Definición:**  
Un esquema es una descripción abstracta de algo, como la estructura de una API o la forma de los datos. En FastAPI, el esquema describe las rutas, operaciones y tipos de datos de la API.

**Ejemplo:**

```json
{
  "openapi": "3.1.0",
  "info": {
    "title": "FastAPI",
    "version": "0.1.0"
  },
  "paths": {
    "/items/": {
      "get": {
        "responses": {
          "200": {
            "description": "Successful Response"
          }
        }
      }
    }
  }
}
```

**Descripción del ejemplo:**  
Este es un fragmento del esquema OpenAPI generado por FastAPI. Describe la ruta `/items/` y la operación GET asociada.

---

## L - Async/Await

**Definición:**  
`async` y `await` son palabras clave en Python que permiten manejar operaciones asíncronas. FastAPI soporta funciones asíncronas para manejar solicitudes de manera eficiente.

**Ejemplo:**

```python

@app.get("/async-example/")
async def async_example():
    return {"message": "Esto es asíncrono!"}
```

**Descripción del ejemplo:**  
La función `async_example()` es asíncrona, lo que significa que puede manejar múltiples solicitudes simultáneamente sin bloquear el servidor.
