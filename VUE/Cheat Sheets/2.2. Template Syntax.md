## A - Template Syntax

**Definición:** La forma en que Vue permite enlazar la interfaz de usuario (DOM) con los datos de un componente. Utiliza HTML como base, con extensiones para la reactividad.

**Ejemplo:**

```html
<span>Mensaje: {{ msg }}</span>
```

(El template se actualiza automáticamente cuando cambia la propiedad msg del componente.)

## B - Text Interpolation

**Definición:** La forma más básica de mostrar datos en el template, usando la sintaxis de "Mustache" (doble llave): `{{ ... }}`.

**Ejemplo:**

```html
<span>El valor es: {{ miValor }}</span>
```

(El contenido dentro de las llaves se reemplaza con el valor de la propiedad miValor.)

## C - Raw HTML

**Definición:** Para mostrar HTML real en lugar de texto plano, se usa la directiva v-html.

**Ejemplo:**

```html
<p>Usando v-html: <span v-html="htmlContent"></span></p>
```

(El contenido de htmlContent se interpreta como HTML.)

Advertencia de seguridad: Usar v-html con contenido no confiable puede llevar a vulnerabilidades XSS.

## D - Attribute Bindings

**Definición:** Para enlazar atributos HTML a datos, se usa la directiva v-bind o su abreviatura :.

**Ejemplo:**

```html
<div v-bind:id="dynamicId"></div>
<div :class="dynamicClass"></div>
```

(El atributo id se actualiza con el valor de dynamicId, y class con dynamicClass.)

## E - Shorthand for v-bind

**Definición:** La abreviatura : es una forma más concisa de usar v-bind.

**Ejemplo:**

```html
<div :id="dynamicId"></div>
```

(Es equivalente a `<div v-bind:id="dynamicId"></div>`.)

## F - Same-name Shorthand

**Definición:** (Vue 3.4+) Si el nombre del atributo es el mismo que el valor de JavaScript, se puede omitir el valor.

**Ejemplo:**

```html
<div :id></div>
<!-- Equivalente a :id="id" -->
```

(Simplifica la sintaxis cuando el nombre del atributo y la variable son iguales.)

## G - Boolean Attributes

**Definición:** Los atributos booleanos (como disabled) se incluyen si el valor enlazado es verdadero o una cadena vacía. Se omiten para otros valores falsos.

**Ejemplo:**

```html
<button :disabled="isButtonDisabled">Botón</button>
```

(El botón estará deshabilitado si isButtonDisabled es verdadero.)

## H - Dynamically Binding Multiple Attributes

**Definición:** Se puede enlazar un objeto JavaScript a múltiples atributos usando v-bind sin argumento.

**Ejemplo:**

```Js
const objectOfAttrs = {
  id: 'container',
  class: 'wrapper'
}
```

```html
<div v-bind="objectOfAttrs"></div>
```

(El div tendrá los atributos id y class definidos en el objeto.)

## I - Using JavaScript Expressions

**Definición:** Se pueden usar expresiones JavaScript dentro de las interpolaciones de texto y los valores de las directivas.

**Ejemplo:**

```html
{{ number + 1 }} {{ ok ? 'Sí' : 'No' }}
<div :id="`item-${id}`"></div>
```

(Las expresiones se evalúan en el contexto del componente.)

## J - Expressions Only

**Definición:** Cada enlace solo puede contener una expresión, no sentencias.

**Ejemplo:**

```html
<!-- Esto NO funciona -->
{{ var a = 1 }} {{ if (ok) { return message } }}
```

(Solo se permiten expresiones que retornen un valor.)

## K - Calling Functions

**Definición:** Se pueden llamar métodos del componente dentro de las expresiones de enlace.

**Ejemplo:**

```html
<time :title="formatDate(date)">{{ date }}</time>
```

(El método formatDate se llama para formatear la fecha.)

Importante: Las funciones llamadas en enlaces deben ser puras y sin efectos secundarios.

## L - Restricted Globals Access

**Definición:** Las expresiones de template tienen acceso limitado a variables globales. Se pueden agregar más a través de app.config.globalProperties.

**Ejemplo:**

```Js
// En el componente
app.config.globalProperties.miGlobal = 'Hola'
```

```html
{{ miGlobal }}
<!-- Funciona -->
```

(Las variables globales personalizadas deben definirse explícitamente.)

## M - Directives

**Definición:** Atributos especiales con el prefijo v- que aplican comportamiento reactivo al DOM.

**Ejemplo:**

```html
<p v-if="isVisible">Visible</p>
```

(El elemento p se muestra u oculta según el valor de isVisible.)

## N - Arguments in Directives

**Definición:** Algunas directivas pueden tener un "argumento" después de dos puntos, que especifica a qué atributo o evento se aplica la directiva.

**Ejemplo:**

```html
<a v-bind:href="url">Enlace</a> <button v-on:click="handleClick">Click</button>
```

(El argumento href en v-bind y click en v-on.)

## O - Shorthand for v-on

**Definición:** La abreviatura @ es una forma más concisa de usar v-on.

**Ejemplo:**

```html
<button @click="handleClick">Click</button>
```

(Es equivalente a `<button v-on:click="handleClick">Click</button>`.)

## P - Dynamic Arguments

**Definición:** Se pueden usar expresiones JavaScript en los argumentos de las directivas, encerrándolas entre corchetes.

**Ejemplo:**

```html
<a :[attributeName]="url">Enlace</a>
<button @[eventName]="handleEvent">Click</button>
```

(El atributo y el evento se determinan dinámicamente.)

## Q - Dynamic Argument Value Constraints

**Definición:** Los argumentos dinámicos deben evaluarse a una cadena o null. Otros valores no son válidos.

**Ejemplo:**

```html
<div :[123]="value"></div>
<!-- Error -->
<div :[null]="value"></div>
<!-- Válido -->
```

(Solo se permiten cadenas o null como valores de argumentos dinámicos.)

## R - Dynamic Argument Syntax Constraints

**Definición:** Los argumentos dinámicos tienen restricciones de sintaxis. No se permiten espacios ni comillas dentro de los corchetes.

**Ejemplo:**

```html
<div :['foo' + bar]="value"></div> <!-- Error -->
```

(Para argumentos complejos, se recomienda usar propiedades computadas.)

## S - Modifiers

**Definición:** Postfijos especiales con un punto que indican que una directiva debe enlazarse de forma especial.

**Ejemplo:**

```html
<form @submit.prevent="onSubmit">...</form>
```

(El modificador .prevent llama a event.preventDefault().)
