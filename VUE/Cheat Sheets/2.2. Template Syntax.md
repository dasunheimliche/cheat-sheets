## A - Template Syntax

**Definición:** La sintaxis de plantilla de Vue es una manera de escribir HTML con esteroides. Te permite conectar visualmente el HTML que ves en pantalla (el DOM) con los datos que viven en tu componente de Vue. Piensa en ello como un "puente" que hace que tu interfaz reaccione a los cambios en tus datos. Usa HTML válido que es transformado por Vue en código JavaScript súper eficiente. En lugar de manipular el DOM directamente, usas la plantilla de Vue, ¡y Vue se encarga de todo el trabajo duro!. **Analogía de la vida cotidiana:** Imagina que tu template es una maqueta arquitectónica que se ajusta automáticamente a medida que cambian las necesidades del cliente (tus datos), Vue es el arquitecto que realiza las modificaciones sin que tengas que reconstruir toda la maqueta.

## B - Text Interpolation

**Definición:** Es la forma más simple de mostrar datos dentro de tu HTML. Utiliza dobles llaves `{{ }}` (llamadas "Mustaches") para "inyectar" valores de datos dentro del texto. Vue reemplazará automáticamente esas llaves con los valores correspondientes y mantendrá esos valores actualizados. **Analogía de la vida cotidiana:** Como si en un formulario en un pdf o word hubieses agregado "campos" que se llenan automáticamente con los datos de una base de datos.

**Ejemplo:**

```language
<template>
  <span>El mensaje es: {{ message }}</span>
</template>
<script setup>
import { ref } from 'vue';
const message = ref('Hola Mundo!');
</script>
```

**Descripción del ejemplo:** Aquí, `{{ message }}` muestra el valor de la variable `message` del componente. Si `message` cambia, el texto en el `<span>` se actualizará de forma automática. El `ref` de `vue` crea una variable reactiva en javascript.

## C - Raw HTML

**Definición:** Permite mostrar código HTML en lugar de texto plano, utilizando la directiva `v-html`. Si lo intentas directamente con las dobles llaves `{{ }}`, verás el código HTML en sí mismo como texto plano, con `v-html`, el HTML se renderiza. OJO, ¡uso peligroso!: `v-html` debe usarse _solo_ en contenido confiable para evitar posibles agujeros de seguridad XSS. **Analogía de la vida cotidiana:** Piensa que `{{ }}` es como imprimir una página web que no tiene conexión con internet y `v-html` como tenerla en la pestaña del navegador lista para usarse con imagenes y todo el contenido que requiera HTML.

**Ejemplo:**

```language
<template>
  <p>Con texto simple: {{ rawHtml }}</p>
  <p>Con v-html: <span v-html="rawHtml"></span></p>
</template>
<script setup>
import { ref } from 'vue';

const rawHtml = ref('<span style="color: red">¡Esto es rojo!</span>');
</script>
```

**Descripción del ejemplo:** El primer `<p>` muestra el código HTML como texto, mientras que el segundo `<p>` usa `v-html` para interpretar el HTML como tal, resultando en texto rojo. Recuerda, NO lo uses para componer vistas parciales, para eso, crea componentes.

## D - Attribute Bindings

**Definición:** Utilizas `v-bind:` (o su atajo `:`) para enlazar un atributo HTML al valor de una propiedad o una expresión. Esto es vital ya que las dobles llaves no sirven para atributos HTML. Vue se asegura de mantener el atributo siempre en sincronía con su dato correspondiente. **Analogía de la vida cotidiana:** Como conectar un enchufe a la corriente; El `v-bind` es el enchufe, y cada que haya un cambio en la corriente (la data) lo verá el artefacto conectado.

**Ejemplo:**

```language
<template>
  <div :id="dynamicId">Este es un div con ID dinamico.</div>
</template>
<script setup>
import { ref } from 'vue';

const dynamicId = ref('mi-div-unico');
</script>
```

**Descripción del ejemplo:** Aquí, el `id` del `<div>` se actualiza si `dynamicId` cambia. Vue lo mantiene todo sincronizado.

## E - Shorthand

**Definición:** Un "atajo" es la forma abreviada de escribir la directiva `v-bind:`. En lugar de escribir `v-bind:algún-atributo`, simplemente usamos `:algún-atributo`. Hace tu código más limpio y rápido de escribir. Es una característica común en Vue, ¡así que familiarízate! **Analogía de la vida cotidiana:** Imagina que usar "por favor" como una abreviación para una frase completa como: "Seria tan amable de hacerme un favor" es una forma común de ahorrarte letras sin perder la intención

**Ejemplo:**

```language
<template>
  <a :href="link" :class="{active: isActive}"> Enlace dinamico</a>
</template>
<script setup>
import { ref } from 'vue';
const link = ref('https://google.com')
const isActive= ref(true)
</script>
```

**Descripción del ejemplo:** Aquí, `:href` es igual que `v-bind:href`, y lo mismo con `class`, el códgio es más limpio y sencillo de entender

## F - Same-Name Shorthand

**Definición:** Es una "súper" abreviatura. Cuando un atributo tiene el mismo nombre que su valor a enlazar, puedes omitir el valor de ese atributo. Por ejemplo, `:id` equivale a `:id="id"`. Esta "súper abreviatura" está disponible a partir de la versión 3.4+ de Vue. **Analogía de la vida cotidiana:** Cuando en la lista del supermercado escribes solo "manzana" sabiendo que te referes al precio o la cantidad que deseas

**Ejemplo:**

```language
<template>
  <div :id>Esto es un div con un ID implicito.</div>
</template>
<script setup>
import { ref } from 'vue';
const id = ref('mi-div-id')
</script>
```

**Descripción del ejemplo:** `:id` es equivalente a `:id="id"`, siempre que en el `script` haya una variable llamada `id` . Más rápido y fácil.

## G - Boolean Attributes

**Definición:** Estos atributos (como `disabled` o `readonly`) indican "verdadero" cuando están presentes y "falso" cuando NO están. Con `v-bind`, si su valor asociado es "truthy", el atributo aparecerá. Con un valor "falsy", el atributo desaparece. Si quieres mantener la consistencia con HTML normal puedes poner una cadena vacía a una propiedad como `disabled=""`, de ese modo siempre estará presente y mantendrá el mismo comportamiento como si le pasaras una cadena truthy o directamente true **Analogía de la vida cotidiana:** como cuando apretas el interruptor de luz para encenderla (true) y lo sueltas para apagarla(false) , sin ese switch (atributo booleano) no es posible encenderla o apagarla, este comportamiento también aplica a un checkbox de una web donde true marca y false desmarca la casilla

**Ejemplo:**

```language
<template>
  <button :disabled="isDisabled">Haz Clic</button>
</template>
<script setup>
import { ref } from 'vue';

const isDisabled = ref(true);
</script>
```

**Descripción del ejemplo:** Si `isDisabled` es verdadero, el botón estará desactivado. Cambia `true` por `false` para activar el boton.

## H - Dynamically Binding Multiple Attributes

**Definición:** `v-bind` sin argumentos (solo `v-bind`) se usa para enviar todos los pares clave-valor de un objeto Javascript directamente a los atributos de un elemento HTML. Cada clave se convertirá en el nombre de un atributo. Súper útil si los atributos son dinamicos y están en un objeto. **Analogía de la vida cotidiana:** Imagina una caja de herramientas que incluye diferentes llaves de un solo golpe puedes pegarla a los pernos de las piezas con solo una conexión o con un golpe en la caja.

**Ejemplo:**

```language
<template>
  <div v-bind="atributos">Hola</div>
</template>
<script setup>
import { ref } from 'vue';

const atributos = ref({
  id: 'container',
  class: 'box',
  style: 'background-color:yellow'
});
</script>
```

**Descripción del ejemplo:** El div toma `id`, `class`, y `style` del objeto `atributos` con un único `v-bind`. El codigo es mucho mas sencillo si los valores de un elemento se guardan previamente

## I - Using JavaScript Expressions

**Definición:** Vue te deja usar Javascript dentro de tus templates: sumas, operaciones ternarias o llamadas a métodos. ¡Mucho poder en el código que genera tu frontend!. Vue evaluará la expresión Javascript, en el contexto del componente. Pero ¡ojo!, una regla vital: _cada enlace debe ser una sola expresión_ Javascript. Evita condicionales tipo `if`, puedes usar una expresión ternaria. Por "expresión" se entiende algo que puede ir luego de un `return` en JavaScript. **Analogía de la vida cotidiana:** Como cuando pides una pizza con extra de queso y pepperoni. "Extra de queso + peperoni" seria la expesion que genera la combinacion en tu pizza, es un unico elemento que a su vez combina o genera otras cosas.

**Ejemplo:**

```language
<template>
  <p>{{ 2 * counter }}</p>
  <p>{{ active ? 'Sí' : 'No' }}</p>
  <div :class="{ active: active }">Soy activo?</div>
</template>
<script setup>
import { ref } from 'vue';

const counter = ref(5)
const active = ref(true)
</script>
```

**Descripción del ejemplo:** En la primera línea calcula una multiplicación, en la segunda se evalua una expresión ternaria y en el tercero usamos `class` en base al estado de una variable. Es un codigo compacto pero muy poderoso.

## J - Calling Functions

**Definición:** También puedes llamar funciones del componente dentro de tus templates. Importante, estas funciones deben ser puros calculos, que NO cambien datos ni usen `async` ¡Deben evitar tener efectos secundarios, especialmente cuando los datos de componente se actualicen, la funcion va a correr!. Las funciones que se retornen, se pueden considerar datos del componente y los renderizados reaccionaran a estos cambios. **Analogía de la vida cotidiana:** como si usases una calculadora para evaluar precios o montos rapidamente a partir de formulas fijas, si los precios varían, debes ejecutar la operacion nuevamente pero nunca modificas los precios desde la calculadora.

**Ejemplo:**

```language
<template>
  <p>El precio es: {{ calcularPrecio(price,tax) }}</p>
</template>
<script setup>
import { ref } from 'vue';

const price = ref(10)
const tax = ref (0.15)

function calcularPrecio(price,tax){
    return price * (1+tax)
}
</script>
```

**Descripción del ejemplo:** Se calcula el precio dentro del componente, esta funión debe estar en el scope del componente. No te compliques , sencillito para la vista del usuario.

## K - Restricted Globals Access

**Definición:** Vue restringe el acceso a globales dentro de tus templates. Solo puedes usar variables como `Math`, `Date` y algunas built-in. No puedes meterte directamente con objetos de la `window`, por seguridad. Para habilitar una variable a nivel global puedes agregar `app.config.globalProperties`. ¡Menos sorpresas con alcance de variables!. Esta restricción no implica que una propiedad no este accesible por ejemplo a un array que contenga un objeto puedes hacer algo como `myarray[0].price` y va a funcionar correctamente. **Analogía de la vida cotidiana:** Piensa en tu barrio cerrado, a los parques y demas areas puedes ir y disfrutar sin inconvenientes, pero a algunas habitaciones privadas del interior de una casa por privacidad no tienes acceso, aunque compartas zonas en común con ellos.

**Ejemplo:**

```language
<template>
  <p>Fecha actual: {{  new Date().toLocaleDateString()  }}</p>
  <p>Numero aleatorio: {{ Math.random() }} </p>
</template>
```

**Descripción del ejemplo:** En la plantilla accedemos a metodos de `Date` y `Math`. Vue es claro en qué métodos puedes usar. Si necesitaramos acceder a la global `window` podemos agregar `app.config.globalProperties.$window = window;` para utilizar `$window` en el template con la global `window`.

## L - Directives

**Definición:** Son atributos HTML especiales con prefijo `v-`. Le dicen a Vue qué debe hacer con elementos del DOM. Por ejemplo: `v-if` controla si un elemento se muestra o no, `v-bind` conecta el DOM con un dato y `v-html` renderiza codigo html Las directivas están "pendientes" para que al cambiar el dato asociado reaccionen los cambios correspondientes al DOM. **Analogía de la vida cotidiana:** Son como "instrucciones especiales" en la plantilla de la receta que modifican como se usa un ingrediente en el platillo final. Si usas `v-if` "condiciona el uso de un ingrediente", si usas `v-html` es como si fuera "el emplatado final y único del platillo", las directivas cambian como tu platillo luce de la mano de las recetas.

**Ejemplo:**

```language
<template>
  <p v-if="isVisible"> ¡Ahora me ves! </p>
  <div v-html="htmlCode"></div>
  <a :href="link"> Enlace</a>
</template>
<script setup>
import { ref } from 'vue';
const isVisible = ref(true)
const htmlCode = ref('<p style="color:green;">Soy un codigo renderizado desde html.</p>')
const link = ref('https://vuejs.org')
</script>
```

**Descripción del ejemplo:** Usa `v-if`, `v-html` y el shorthand `:href`. Directivas en acción. Control total del DOM!.

## M - Arguments

**Definición:** Los argumentos son pequeños textos, despues de un `:`, en una directiva ( ejemplo en `v-bind:atributo` seria "atributo") que especifican de manera precisa a que parte del elemento HTML queremos impactar. Con `v-bind` defines un atributo ,con `v-on` qué evento vas a escuchar. Estos son solo algunos ejemplos, las directivas te dejan hacer cambios especificos donde las necesites. **Analogía de la vida cotidiana:** Seria como decirle al mesero qué clase de refresco te gustaria en el almuerzo `Coca-Cola` es el argumento al refresco así tu pides refresco `:` + marca

**Ejemplo:**

```language
<template>
  <a v-bind:href="url"> ... </a>
  <a @click="showMensaje">...</a>
</template>
<script setup>
import { ref } from 'vue';
const url = ref('https://vuejs.org')
const showMensaje = () => alert("Me hiciste click!");
</script>
```

**Descripción del ejemplo:** Aquí `href` es el argumento de `v-bind`. `click` el argumento de `@`. El elemento se vera afectado especificamente por su argumento en cada directiva.

## N - Dynamic Arguments

**Definición:** Te permiten que el _nombre_ del argumento de una directiva sea dinámico, con JavaScript! Solo usa `[]` alrededor de tu variable, como en `:[nombre]` o `v-bind:[nombre]` , Vue entenderá que esa es una expresión a evaluar, así tienes mucha flexibilidad!. **Analogía de la vida cotidiana:** Sería como el control remoto universal para controlar TV y demas aparatos. Cada vez que seleccionas en el display un nuevo gadget que vas a usar esa selecciion es dinamica para el mando universal.

**Ejemplo:**

```language
<template>
  <a v-bind:[atributo]="url"> ... </a>
  <a @[evento]="saludo"> ... </a>
</template>
<script setup>
import { ref } from 'vue';
const atributo = ref('href')
const evento = ref('click')
const url = ref('https://vuejs.org')
const saludo = ()=> alert('Hola!!')
</script>
```

**Descripción del ejemplo:** `atributo` y `evento` son valores dinámicos para el `href` en el `v-bind`, y el evento del `@`. Es un código muy versátil que permite gran capacidad para los template y componente

## O - Dynamic Argument Value Constraints

**Definición:** Cuando los argumentos son dinámicos, su valor debe ser _string_. `Null` se acepta para eliminar la asignación. Otro tipo de dato da error, Vue es estricto con esto. Si quieres valores complejos para un atributo usa un `computed`, pero es para casos específicos. **Analogía de la vida cotidiana:** En los juegos solo puedes acceder a objetos o partes si los seleccionas. Solo debes usar texto como argumento ( como seleccionar un nivel ) por el nombre o usar un identificador numero siempre sera como texto de manera directa al input o campo de la sección en el juego.

**Ejemplo:**

```language
<template>
   <a v-bind:[attribute]="value"  > Hola, soy un link</a>
</template>
<script setup>
import { ref } from 'vue';

const attribute=ref('href')
const value= ref('https://google.com')

</script>
```

**Descripción del ejemplo:** Si bien las propiedades `attribute` y `value` son ref's, se parsean como un `string` al momento de usarlos para definir dinámicamente los argumentos o propiedades de la plantilla. En cambio con `null` el argumento dinámico `[attribute]` se elimina como el siguiente ejemplo:

```language
<template>
  <a v-bind:[attribute]="value"  > Hola, soy un link</a>
  <button @click="changeAttribute">Click me for delete the attribute</button>
</template>
<script setup>
import { ref } from 'vue';

const attribute=ref('href')
const value= ref('https://google.com')

function changeAttribute() {
  attribute.value = null;
}

</script>
```

**Descripción del ejemplo:** Al darle al click el link va a cambiar el atributo a `null` e internamente ese atributo se remueve del elemento HTML `a`, no muestra error por este comportamiento específico

## P - Dynamic Argument Syntax Constraints

**Definición:** Con los argumentos dinámicos, evita ciertos caracteres: espacios, comillas, y uppercase( solo en algunos casos como "templates in dom") en nombre del atributo HTML, pues pueden generar conflicto en el parseado. Para lógica compleja usa un `computed` y evita estos dolores de cabeza!. **Analogía de la vida cotidiana:** Cuando quieres escribir la direccion completa de tu casa puedes tener dificultades si en esa dirección tienes comillas dobles (") y espacios en la direccion es mejor omitirlos a escribirlos si no es relevante al contexto en general.

**Ejemplo:**

```language
<!-- This will trigger a compiler warning. -->
<template>
 <a :['foo' + bar]="value"> ... </a>
</template>
```

**Descripción del ejemplo:** En este ejemplo se genera un error por comillas en `['foo' + bar]` para definir un atributo

## Q - Modifiers

**Definición:** Los modificadores son postfijos a las directivas (por ejemplo, en `v-on:click.prevent` seria `.prevent` ) que cambian cómo funcionan esas directivas. Por ejemplo: `.prevent` en `@submit` evita que el formulario recargue la página, usando el `event.preventDefault()`. Es la cereza que completa la acción principal de la directiva. **Analogía de la vida cotidiana:** Es como agregar un "filtro" a la orden. Si pides un café `@`, el modificador sería `.con leche`, `.`+ modificado cambia a cómo recibes la orden principal, pero manteniéndola presente como cafe.

**Ejemplo:**

```language
<template>
 <form @submit.prevent="enviar">
    ...
    </form>
  </template>
 <script setup>
function enviar(){
alert("No recarga el formulario!");
 }
</script>
```

**Descripción del ejemplo:** `.prevent` hace que el formulario NO re-cargue, el form, por tanto ejecuta solo la función, modificando la funcionalidad del `@submit`
