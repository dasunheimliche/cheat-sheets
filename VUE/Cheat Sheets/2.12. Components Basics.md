## A - Components

**Definición:** Piezas independientes y reutilizables de la interfaz de usuario. Permiten encapsular contenido y lógica.

**Ejemplo:**

Un componente ButtonCounter que muestra un botón y un contador.

```Vue
<!-- ButtonCounter.vue -->
<script setup>
import { ref } from 'vue';
const count = ref(0);
</script>

<template>
  <button @click="count++">Clics: {{ count }}</button>
</template>
```

(Los componentes permiten organizar la interfaz de usuario en partes más pequeñas y manejables.)

## B - Defining a Component (SFC)

**Definición:** Se define un componente en un archivo .vue (Single-File Component) con `<script>`, `<template>` y `<style>` (opcional).

**Ejemplo:**

```Vue
<!-- MyComponent.vue -->
<script setup>
import { ref } from 'vue';
const message = ref('Hola');
</script>

<template>
  <p>{{ message }}</p>
</template>
```

(Un componente simple que muestra un mensaje.)

## C - Defining a Component (JS Object)

**Definición:** Se define un componente como un objeto JavaScript con opciones como setup() y template.

**Ejemplo:**

```Js
import { ref } from 'vue';

export default {
  setup() {
    const count = ref(0);
    return { count };
  },
  template: `
    <button @click="count++">
      Clics: {{ count }}
    </button>
  `,
};
```

(Un componente definido como un objeto JavaScript.)

## D - Using a Component

**Definición:** Se importa un componente hijo en el componente padre y se usa en el template.

**Ejemplo:**

```Vue
<!-- ParentComponent.vue -->
<script setup>
import ButtonCounter from './ButtonCounter.vue';
</script>

<template>
  <h1>Componente Padre</h1>
  <ButtonCounter />
</template>
```

(El componente ButtonCounter se usa dentro de ParentComponent.)

## E - Component Instances

**Definición:** Cada vez que se usa un componente, se crea una nueva instancia con su propio estado.

**Ejemplo:**

```Vue
<!-- ParentComponent.vue -->
<template>
  <ButtonCounter />
  <ButtonCounter />
</template>
```

(Cada ButtonCounter tiene su propio contador.)

## F - Passing Props

**Definición:** Se pasan datos a un componente hijo a través de atributos personalizados llamados props. Se declaran con defineProps en `<script setup>`.

**Ejemplo:**

```Vue
<!-- BlogPost.vue -->
<script setup>
defineProps(['title']);
</script>

<template>
  <h4>{{ title }}</h4>
</template>
```

```Vue
<!-- ParentComponent.vue -->
<template>
  <BlogPost title="Mi primer post" />
</template>
```

(El componente BlogPost recibe el título a través de la prop title.)

## G - Dynamic Props

**Definición:** Se pueden pasar valores dinámicos a las props usando v-bind o su abreviatura :.

**Ejemplo:**

```Vue
<!-- ParentComponent.vue -->
<script setup>
import { ref } from 'vue';
const posts = ref([{ id: 1, title: 'Post 1' }]);
</script>

<template>
  <BlogPost v-for="post in posts" :key="post.id" :title="post.title" />
</template>
```

(La prop title se enlaza dinámicamente a la propiedad title de cada objeto en el array posts.)

## H - Listening to Events

**Definición:** El componente padre puede escuchar eventos emitidos por el componente hijo usando v-on o su abreviatura @.

**Ejemplo:**

```Vue
<!-- BlogPost.vue -->
<template>
  <button @click="$emit('enlarge-text')">Aumentar texto</button>
</template>
```

```Vue
<!-- ParentComponent.vue -->
<template>
  <BlogPost @enlarge-text="fontSize += 0.1" />
</template>
```

(El componente padre escucha el evento enlarge-text y actualiza fontSize.)

## I - Emitting Events

**Definición:** El componente hijo emite eventos usando $emit o la función retornada por defineEmits.

**Ejemplo:**

```Vue
<!-- BlogPost.vue -->
<script setup>
const emit = defineEmits(['enlarge-text']);
function handleClick() {
  emit('enlarge-text');
}
</script>

<template>
  <button @click="handleClick">Aumentar texto</button>
</template>
```

(El componente hijo emite el evento enlarge-text al hacer clic en el botón.)

## J - Declaring Emitted Events

**Definición:** Se pueden declarar los eventos que emite un componente usando defineEmits en `<script setup>`.

**Ejemplo:**

```Vue
<!-- BlogPost.vue -->
<script setup>
defineEmits(['enlarge-text']);
</script>
```

(Se declara que el componente emite el evento enlarge-text.)

## K - Content Distribution with Slots

**Definición:** Se pueden pasar elementos HTML a un componente usando `<slot>`.

**Ejemplo:**

```Vue
<!-- AlertBox.vue -->
<template>
  <div class="alert-box">
    <strong>Error:</strong>
    <slot />
  </div>
</template>
```

```Vue
<!-- ParentComponent.vue -->
<template>
  <AlertBox>
    Algo salió mal.
  </AlertBox>
</template>
```

(El contenido "Algo salió mal." se inserta en el `<slot>` del componente AlertBox.)

## L - Dynamic Components

**Definición:** Se puede cambiar dinámicamente el componente renderizado usando el elemento `<component>` y el atributo :is.

**Ejemplo:**

```Vue
<template>
  <component :is="currentTab"></component>
</template>

<script setup>
import { ref } from 'vue';
import TabA from './TabA.vue';
import TabB from './TabB.vue';
const currentTab = ref(TabA);
</script>
```

(El componente renderizado cambia según el valor de currentTab.)

## M - In-DOM Template Parsing Caveats

**Definición:** Limitaciones al usar templates directamente en el DOM:

Case Insensitivity: Los nombres de componentes y props deben ser kebab-case.

Self Closing Tags: Se deben usar etiquetas de cierre explícitas.

Element Placement Restrictions: Algunos elementos tienen restricciones sobre qué elementos pueden contener.

**Ejemplo:**

```Html
<!-- Template en el DOM -->
<blog-post post-title="Hola" @update-post="onUpdate"></blog-post>
```

(Se usa kebab-case para el nombre del componente y la prop.)

## N - is Attribute for Element Placement

**Definición:** Se usa el atributo is para renderizar componentes dentro de elementos con restricciones de contenido.

**Ejemplo:**

```Html
<table>
  <tr is="vue:blog-post-row"></tr>
</table>
```

(Se usa is="vue:blog-post-row" para renderizar el componente blog-post-row dentro de un `<tr>`.)
