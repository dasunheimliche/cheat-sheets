## B - Crear tu Primera Aplicación Vue (¡Ahora en Serio!)

#### 1. **Definición:**

Esta es la forma recomendada para empezar un proyecto Vue desde cero en tu computadora. Usaremos una herramienta oficial que nos crea la estructura básica del proyecto, lista para funcionar con las características modernas de Vue. A esto se le llama "scaffolding" (crear el andamiaje inicial).

**Requisitos previos:**

- Sentirte cómodo/a usando la línea de comandos (la terminal o consola).
- Tener instalado [Node.js](https://nodejs.org/) (versión 18.3 o superior). Node.js es un entorno que permite ejecutar JavaScript fuera del navegador y trae consigo `npm`, un gestor de paquetes que usaremos.

#### 2. **Ejemplo:**

Abre tu terminal, navega hasta la carpeta donde quieras crear tu proyecto y ejecuta este comando (sin el `$`, que solo representa el prompt de la terminal):

```bash
# Usando npm (viene con Node.js)
$ npm create vue@latest
```

_También puedes usar otros gestores de paquetes si los prefieres:_

```bash
# Usando pnpm
$ pnpm create vue@latest

# Usando yarn (v1+)
$ yarn create vue

# Usando yarn (v2+)
$ yarn create vue@latest

# Usando bun
$ bun create vue@latest
```

**Explicación del ejemplo:**
Este comando descarga y ejecuta `create-vue`, la herramienta oficial. Te hará algunas preguntas para configurar tu proyecto (nombre del proyecto, si quieres usar TypeScript, soporte para pruebas, etc.). Si no estás seguro/a, simplemente presiona `Enter` para elegir la opción por defecto (normalmente "No").

```
✔ Project name: … <tu-nombre-de-proyecto>
✔ Add TypeScript? … No / Yes
✔ Add JSX Support? … No / Yes
... y más preguntas ...

Scaffolding project in ./<tu-nombre-de-proyecto>...
Done.
```

#### 3. **Notas o advertencias:**

- Este método configura un proyecto que usa **Vite** como herramienta de construcción (build tool). Vite es súper rápido y moderno.
- Te permitirá usar **Componentes de Archivo Único** (Single-File Components o SFCs), que son archivos `.vue` donde mezclas HTML, CSS y JavaScript de forma organizada. ¡Son la base de la mayoría de los proyectos Vue!
- Este tipo de proyecto se conoce como **Single Page Application (SPA)**, donde la aplicación carga una sola página HTML y Vue se encarga de actualizar el contenido dinámicamente.

## C - ¡A Correr! Iniciando el Servidor de Desarrollo

#### 1. **Definición:**

Una vez que la herramienta `create-vue` ha creado la carpeta de tu proyecto (como vimos en el **Concepto B**), necesitas "entrar" en esa carpeta, instalar las dependencias (librerías que tu proyecto necesita) y arrancar un servidor local para ver tu aplicación en el navegador mientras desarrollas.

#### 2. **Ejemplo:**

Sigue estos pasos en tu terminal, justo después de crear el proyecto:

```bash
# 1. Entra en la carpeta de tu proyecto (cambia <tu-nombre-de-proyecto> por el nombre real)
$ cd <tu-nombre-de-proyecto>

# 2. Instala las dependencias (usando npm)
$ npm install

# 3. Inicia el servidor de desarrollo
$ npm run dev
```

_Si usaste otro gestor de paquetes para crear el proyecto, úsalo también aquí:_

```bash
# Con pnpm
$ cd <tu-nombre-de-proyecto>
$ pnpm install
$ pnpm run dev

# Con yarn
$ cd <tu-nombre-de-proyecto>
$ yarn
$ yarn dev

# Con bun
$ cd <tu-nombre-de-proyecto>
$ bun install
$ bun run dev
```

**Explicación del ejemplo:**

- `cd <tu-nombre-de-proyecto>`: Cambia tu ubicación actual en la terminal a la carpeta del proyecto.
- `npm install` (o `pnpm install`, `yarn`, `bun install`): Descarga todas las librerías necesarias que se listan en el archivo `package.json` de tu proyecto.
- `npm run dev` (o `pnpm run dev`, `yarn dev`, `bun run dev`): Ejecuta un script (definido en `package.json`) que inicia el servidor de desarrollo de Vite.

La terminal te mostrará una dirección local (algo como `http://localhost:5173/`). ¡Abre esa dirección en tu navegador y verás tu primera aplicación Vue funcionando!

#### 3. **Notas o advertencias:**

- Este servidor de desarrollo es genial porque tiene "Hot Module Replacement" (HMR), lo que significa que cuando guardas cambios en tu código, la página en el navegador se actualiza automáticamente sin perder el estado actual. ¡Magia! ✨
- Los componentes de ejemplo que verás usan la **Composition API** y `<script setup>`, que es la forma moderna y recomendada de escribir componentes en Vue 3. Existe otra forma llamada **Options API**, más tradicional.

## D - Preparando tu App para el Mundo Real (Build de Producción)

#### 1. **Definición:**

Cuando termines de desarrollar tu aplicación y quieras subirla a un servidor web para que la gente la use (lo que se llama "desplegar a producción"), no subes todo el código de desarrollo. Necesitas crear una versión optimizada, más pequeña y rápida de tu app. A esto se le llama "hacer un build de producción".

#### 2. **Ejemplo:**

Desde la carpeta raíz de tu proyecto en la terminal, ejecuta:

```bash
# Usando npm
$ npm run build
```

_O con otros gestores:_

```bash
# Con pnpm
$ pnpm run build

# Con yarn
$ yarn build

# Con bun
$ bun run build
```

**Explicación del ejemplo:**
Este comando le dice a Vite (la herramienta de construcción) que analice tu código, lo optimice (minifique, combine archivos, etc.) y genere los archivos finales listos para producción.

#### 3. **Notas o advertencias:**

- El resultado de este comando se guardará en una carpeta llamada `dist` dentro de tu proyecto.
- Son los archivos _dentro_ de esta carpeta `dist` los que debes subir a tu hosting o servidor web.
- Para más detalles sobre cómo desplegar, consulta la [Guía de Despliegue a Producción](https://vuejs.org/guide/best-practices/production-deployment.html).

## F - Usar Vue Directamente desde un CDN (Sin Compilación)

#### 1. **Definición:**

¿Y si no quieres (o no puedes) usar Node.js ni herramientas de compilación? ¡No hay problema! Puedes incluir Vue en tu página HTML directamente usando una etiqueta `<script>`, apuntando a un archivo Vue alojado en una **CDN** (Content Delivery Network). Una CDN es una red de servidores que alojan archivos para que se descarguen rápido desde cualquier lugar del mundo.

#### 2. **Ejemplo:**

Añade esta línea en tu archivo HTML, preferiblemente dentro del `<head>` o justo antes de cerrar el `</body>`:

```html
<script src="https://unpkg.com/vue@3/dist/vue.global.js"></script>
```

**Explicación del ejemplo:**
Esto carga la librería Vue (versión 3) desde la CDN `unpkg`. Una vez cargado, tendrás acceso a Vue en tu JavaScript.

#### 3. **Notas o advertencias:**

- Este método es más simple porque no requiere un "paso de compilación" (build step).
- Es ideal para mejorar páginas HTML estáticas existentes o para integrar Vue en aplicaciones hechas con otros frameworks de backend (como Rails, Django, Laravel, etc.).
- **Importante:** Con este método, **no puedes** usar la sintaxis de Componentes de Archivo Único (archivos `.vue`). Tendrás que escribir tus plantillas HTML como strings dentro de tu JavaScript o usar otras técnicas.
- Puedes usar otras CDNs como [jsdelivr](https://www.jsdelivr.com/package/npm/vue) o [cdnjs](https://cdnjs.com/libraries/vue).

## G - El "Modo Global" de Vue desde CDN (con Options API)

#### 1. **Definición:**

Cuando cargas Vue desde la CDN usando el archivo `vue.global.js` (como en el **Concepto F**), Vue se hace disponible como un objeto global llamado `Vue` en tu navegador. Puedes acceder a todas las funciones principales de Vue a través de este objeto. Aquí vemos cómo usarlo con la **Options API**, una forma de organizar el código de tus componentes.

#### 2. **Ejemplo:**

```html
<!-- 1. Incluye Vue desde la CDN -->
<script src="https://unpkg.com/vue@3/dist/vue.global.js"></script>

<!-- 2. Tu HTML con un marcador de Vue -->
<div id="app">{{ message }}</div>

<!-- 3. Tu script para crear la app Vue -->
<script>
  // Accede a createApp desde el objeto global Vue
  const { createApp } = Vue;

  // Crea la aplicación Vue
  createApp({
    // Options API: usa la opción 'data' para definir datos
    data() {
      return {
        message: "¡Hola Vue desde CDN!",
      };
    },
    // Monta la aplicación en el div con id="app"
  }).mount("#app");
</script>
```

[Demo en CodePen >](https://codepen.io/vuejs-examples/pen/QWJwJLp)

**Explicación del ejemplo:**

1.  Cargamos Vue.
2.  Tenemos un `div` con `id="app"`. Dentro, `{{ message }}` es una "interpolación": Vue reemplazará esto con el valor de la variable `message`.
3.  En el script:
    - Obtenemos la función `createApp` del objeto global `Vue`.
    - Llamamos a `createApp` pasándole un objeto de configuración.
    - Usamos la opción `data()`, que debe devolver un objeto con las variables que queremos usar en nuestra plantilla (aquí, `message`).
    - Usamos `.mount('#app')` para decirle a Vue que controle el elemento HTML con `id="app"`.

#### 3. **Notas o advertencias:**

- Este es el enfoque más sencillo para empezar con Vue desde CDN si prefieres la Options API.

## H - El "Modo Global" con Composition API desde CDN

#### 1. **Definición:**

También puedes usar la **Composition API** (la forma más moderna y flexible de organizar la lógica en Vue) cuando cargas Vue desde la CDN con el archivo `vue.global.js`. La diferencia principal es que usarás la opción `setup()` en lugar de `data()`, `methods()`, etc.

#### 2. **Ejemplo:**

```html
<!-- 1. Incluye Vue desde la CDN -->
<script src="https://unpkg.com/vue@3/dist/vue.global.js"></script>

<!-- 2. Tu HTML con un marcador de Vue -->
<div id="app">{{ message }}</div>

<!-- 3. Tu script para crear la app Vue -->
<script>
  // Accede a createApp y ref desde el objeto global Vue
  const { createApp, ref } = Vue;

  // Crea la aplicación Vue
  createApp({
    // Composition API: usa la opción 'setup'
    setup() {
      // 'ref' crea una variable reactiva
      const message = ref("¡Hola Vue con Composition API desde CDN!");

      // Devuelve las variables/funciones que quieres usar en la plantilla
      return {
        message,
      };
    },
    // Monta la aplicación en el div con id="app"
  }).mount("#app");
</script>
```

[Demo en CodePen >](https://codepen.io/vuejs-examples/pen/eYQpQEG)

**Explicación del ejemplo:**

1.  Cargamos Vue.
2.  El HTML es el mismo.
3.  En el script:
    - Obtenemos `createApp` y también `ref` del objeto global `Vue`. `ref` es una función clave de la Composition API para crear variables "reactivas" (que actualizan la vista cuando cambian).
    - Llamamos a `createApp` pasándole un objeto de configuración.
    - Usamos la opción `setup()`. Esta función contiene la lógica de nuestro componente.
    - Creamos la variable reactiva `message` usando `ref()`.
    - La función `setup()` debe devolver un objeto con todo lo que queramos exponer a la plantilla (en este caso, `message`).
    - Usamos `.mount('#app')` igual que antes.

#### 3. **Notas o advertencias:**

- **¡Ojo!** La mayoría de los ejemplos de Composition API que verás en la documentación usan la sintaxis `<script setup>` dentro de archivos `.vue`. Eso requiere herramientas de compilación. Si usas Vue desde CDN _sin_ compilación, tienes que usar la opción `setup()` como en este ejemplo. Consulta la documentación de la [`opción setup()`](https://vuejs.org/api/composition-api-setup.html) para más detalles.

## I - Usando Módulos ES Nativos desde CDN (con Options API)

#### 1. **Definición:**

Los navegadores modernos soportan **Módulos ES** (ES Modules) de forma nativa. Esto significa que puedes usar las sentencias `import` y `export` de JavaScript directamente en el navegador, ¡sin necesidad de herramientas de compilación! Para usar Vue de esta manera desde una CDN, necesitas dos cosas:

1.  Usar una etiqueta `<script type="module">`.
2.  Importar la versión "ESM Browser" de Vue desde la CDN (un archivo diferente al `vue.global.js`).

Aquí vemos cómo hacerlo con la **Options API**.

#### 2. **Ejemplo:**

```html
<!-- 1. Tu HTML con un marcador de Vue -->
<div id="app">{{ message }}</div>

<!-- 2. Tu script como módulo ES -->
<script type="module">
  // Importa createApp desde la versión ESM de Vue en la CDN
  import { createApp } from "https://unpkg.com/vue@3/dist/vue.esm-browser.js";

  // Crea la aplicación Vue (usando Options API)
  createApp({
    data() {
      return {
        message: "¡Hola Vue con Módulos ES (Options API)!",
      };
    },
  }).mount("#app");
</script>
```

[Demo en CodePen >](https://codepen.io/vuejs-examples/pen/VwVYVZO)

**Explicación del ejemplo:**

1.  El HTML es el mismo.
2.  El script ahora tiene `type="module"`.
3.  Usamos `import { createApp } from '...'` para traer solo la función que necesitamos. Fíjate que la URL apunta a `vue.esm-browser.js`.
4.  El resto del código para crear la app con `data()` es igual que en el **Concepto G**, pero ya no usamos el objeto global `Vue`.

#### 3. **Notas o advertencias:**

- Este método requiere un navegador moderno que soporte Módulos ES.
- Es la forma más "moderna" de usar Vue sin un paso de compilación.

## J - Módulos ES Nativos con Composition API desde CDN

#### 1. **Definición:**

Igual que en el **Concepto I**, podemos usar Módulos ES nativos, pero esta vez con la **Composition API**. Importaremos `createApp` y `ref` (o lo que necesitemos) directamente desde la versión ESM de Vue en la CDN.

#### 2. **Ejemplo:**

```html
<!-- 1. Tu HTML con un marcador de Vue -->
<div id="app">{{ message }}</div>

<!-- 2. Tu script como módulo ES -->
<script type="module">
  // Importa createApp y ref desde la versión ESM de Vue en la CDN
  import {
    createApp,
    ref,
  } from "https://unpkg.com/vue@3/dist/vue.esm-browser.js";

  // Crea la aplicación Vue (usando Composition API)
  createApp({
    setup() {
      const message = ref("¡Hola Vue con Módulos ES (Composition API)!");
      return {
        message,
      };
    },
  }).mount("#app");
</script>
```

[Demo en CodePen >](https://codepen.io/vuejs-examples/pen/MWzazEv)

**Explicación del ejemplo:**

1.  El HTML es el mismo.
2.  El script tiene `type="module"`.
3.  Importamos `createApp` y `ref` desde `vue.esm-browser.js`.
4.  El resto del código para crear la app con `setup()` y `ref()` es igual que en el **Concepto H**, pero usando `import` en lugar del objeto global `Vue`.

#### 3. **Notas o advertencias:**

- Nuevamente, requiere un navegador moderno.
- Recuerda la diferencia con `<script setup>` que mencionamos en la nota del **Concepto H**.

## K - ¡Mapas al Rescate! Simplificando Imports con Import Maps

#### 1. **Definición:**

¿Viste que en los ejemplos de Módulos ES (Conceptos **I** y **J**) tuvimos que usar la URL completa de la CDN en la sentencia `import`? ¡Es un poco largo y feo! Los **Import Maps** son una característica relativamente nueva del navegador que te permite "mapear" nombres cortos (como `'vue'`) a URLs completas. Así, en tu código puedes escribir `import { createApp } from 'vue'` como si tuvieras un sistema de compilación, ¡pero sigue funcionando directamente en el navegador!

#### 2. **Ejemplo:**

```html
<!-- 1. Define el Import Map ANTES de cualquier script type="module" -->
<script type="importmap">
  {
    "imports": {
      "vue": "https://unpkg.com/vue@3/dist/vue.esm-browser.js"
      /* Puedes añadir otros mapeos aquí para otras librerías */
    }
  }
</script>

<!-- 2. Tu HTML con un marcador de Vue -->
<div id="app">{{ message }}</div>

<!-- 3. Tu script como módulo ES, ¡ahora con import simplificado! -->
<script type="module">
  // ¡Mira qué limpio! Solo 'vue'
  import { createApp } from "vue"; // (Usando Options API aquí como ejemplo)

  createApp({
    data() {
      return {
        message: "¡Hola Vue con Import Maps!",
      };
    },
  }).mount("#app");
</script>
```

_Puedes hacer lo mismo con Composition API:_

```html
<!-- Import Map (igual que arriba) -->
<script type="importmap">
  ...
</script>

<!-- HTML (igual que arriba) -->
<div id="app">{{ message }}</div>

<!-- Script con Composition API y import simplificado -->
<script type="module">
  import { createApp, ref } from "vue"; // ¡Limpio!

  createApp({
    setup() {
      const message = ref("¡Hola Vue con Import Maps (Composition API)!");
      return { message };
    },
  }).mount("#app");
</script>
```

[Demo Options API con Import Map >](https://codepen.io/vuejs-examples/pen/wvQKQyM)
[Demo Composition API con Import Map >](https://codepen.io/vuejs-examples/pen/YzRyRYM)

**Explicación del ejemplo:**

1.  Añadimos una etiqueta `<script type="importmap">`. Dentro, un JSON define los "imports". Le decimos al navegador: "cuando alguien importe `'vue'`, en realidad busca el archivo en `'https://unpkg.com/vue@3/dist/vue.esm-browser.js'`".
2.  El HTML no cambia.
3.  En nuestro `<script type="module">`, ahora podemos usar `import ... from 'vue'` directamente. ¡Mucho más legible!

#### 3. **Notas o advertencias:**

- **Soporte de Navegadores:** Import Maps es una característica nueva. Asegúrate de que los navegadores de tus usuarios lo soporten (revisa [caniuse.com/import-maps](https://caniuse.com/import-maps)). Por ejemplo, Safari lo soporta desde la versión 16.4.
- **Desarrollo vs. Producción:** Los ejemplos usan la URL de la versión de _desarrollo_ de Vue (`vue.esm-browser.js`). Para producción, deberías usar la versión optimizada (`vue.esm-browser.prod.js`). Consulta la [Guía de Despliegue a Producción sin Herramientas de Compilación](https://vuejs.org/guide/best-practices/production-deployment.html#without-build-tools).
- **Alternativa:** Si buscas algo aún más ligero para mejoras simples de HTML, considera [`petite-vue`](https://github.com/vuejs/petite-vue), que está diseñado para ese propósito.

## L - Dividiendo el Código en Módulos (con CDN y ES Modules)

#### 1. **Definición:**

Cuando tu aplicación crece un poco, incluso usando Vue desde CDN con Módulos ES, querrás organizar tu código en varios archivos JavaScript para que sea más manejable. Puedes crear tus propios módulos (por ejemplo, un archivo por componente) y luego importarlos en tu script principal.

#### 2. **Ejemplo:**

Imagina que tienes estos dos archivos en la misma carpeta:

**`index.html`:**

```html
<!-- (Asegúrate de incluir el Import Map si quieres usar 'vue' directamente, como en el Concepto K) -->
<script type="importmap">
  { "imports": { "vue": "https://unpkg.com/vue@3/dist/vue.esm-browser.js" } }
</script>

<div id="app"></div>

<script type="module">
  import { createApp } from "vue";
  // Importa tu componente desde otro archivo JS
  import MyComponent from "./my-component.js";

  // Usa el componente importado al crear la app
  createApp(MyComponent).mount("#app");
</script>
```

**`my-component.js` (Versión con Options API):**

```javascript
// Exporta el objeto de configuración del componente por defecto
export default {
  data() {
    return { count: 0 };
  },
  // La plantilla HTML como un string
  template: `
    <div>
      Contador es: {{ count }}
      <button @click="count++">Incrementar</button>
    </div>
  `,
};
```

**`my-component.js` (Versión con Composition API):**

```javascript
import { ref } from "vue"; // Necesitas importar 'ref' si usas Composition API

export default {
  setup() {
    const count = ref(0);
    // No olvides devolverlo para usarlo en la plantilla
    return { count };
  },
  template: `
    <div>
      Contador es: {{ count }}
      <button @click="count++">Incrementar</button>
    </div>
  `,
};
```

**Explicación del ejemplo:**

- En `my-component.js`, definimos nuestro componente (ya sea con `data()` o `setup()`) y lo exportamos usando `export default`. La plantilla HTML se escribe como un string dentro de la opción `template`.
- En `index.html`, dentro del `<script type="module">`, usamos `import MyComponent from './my-component.js'` para traer la definición del componente.
- Luego, pasamos `MyComponent` directamente a `createApp()`.

#### 3. **Notas o advertencias:**

- **¡El Problema `file://`!** Si intentas abrir el archivo `index.html` directamente en tu navegador haciendo doble clic (lo que usa el protocolo `file://`), **fallará**. Los Módulos ES, por razones de seguridad, solo funcionan cuando la página se sirve a través del protocolo `http://` o `https://`. ¡No te asustes! La solución viene en el siguiente concepto.
- **Resaltado de Sintaxis para Plantillas:** Escribir HTML dentro de strings de JavaScript puede ser incómodo. Si usas VS Code, puedes instalar la extensión [es6-string-html](https://marketplace.visualstudio.com/items?itemName=Tobermory.es6-string-html) y añadir un comentario `/*html*/` justo antes del string (`template: /*html*/ \`...\`) para obtener resaltado de sintaxis dentro de la plantilla.

## M - El Servidor HTTP Local: Tu Aliado para Módulos ES

#### 1. **Definición:**

Como vimos en la nota del **Concepto L**, los Módulos ES (`import`/`export` nativos en el navegador) no funcionan si abres tu archivo HTML directamente desde tu sistema de archivos (usando `file://`). Necesitas servir tus archivos a través de un servidor web local para que el navegador los acceda usando `http://`.

#### 2. **Ejemplo:**

La forma más sencilla (si tienes Node.js instalado) es usar el paquete `serve`. Abre tu terminal, navega hasta la carpeta donde están tus archivos `index.html` y `my-component.js`, y ejecuta:

```bash
npx serve
```

**Explicación del ejemplo:**

- `npx` es una herramienta que viene con npm (Node.js) y te permite ejecutar paquetes sin tener que instalarlos globalmente.
- `serve` es un paquete que inicia un servidor HTTP estático simple en la carpeta actual.

La terminal te mostrará una dirección local (probablemente `http://localhost:3000` o similar). Abre _esa_ dirección en tu navegador, ¡y ahora tus Módulos ES funcionarán perfectamente!

#### 3. **Notas o advertencias:**

- Esto resuelve el problema de `file://` mencionado en el **Concepto L**.
- Existen muchas otras formas de iniciar un servidor local:
  - Si usas Python, puedes ejecutar `python -m http.server` en la carpeta.
  - Muchos editores de código tienen extensiones para esto, como "Live Server" para VS Code, que además recarga la página automáticamente cuando haces cambios.
- Recuerda la extensión `es6-string-html` mencionada en el **Concepto L** para mejorar la experiencia de escribir plantillas como strings.
