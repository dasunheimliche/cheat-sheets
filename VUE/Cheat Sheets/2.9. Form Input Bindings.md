## A - v-model

**Definición:** Directiva que crea un enlace bidireccional entre un elemento de formulario y un dato reactivo. Simplifica la sincronización de valores.

**Ejemplo:**

```Vue
<template>
  <input v-model="message" placeholder="Escribe algo" />
  <p>Mensaje: {{ message }}</p>
</template>

<script setup>
import { ref } from 'vue';
const message = ref('');
</script>
```

(El valor del input se sincroniza con la variable message.)

## B - v-model on Different Elements

**Definición:** v-model funciona con diferentes tipos de elementos de formulario: `<input>`, `<textarea>`, `<select>`.

**Ejemplo:**

```Html
<input type="text"> y <textarea>: Usan la propiedad value y el evento input.

<input type="checkbox"> y <input type="radio">: Usan la propiedad checked y el evento change.

<select>: Usa la propiedad value y el evento change.
```

**Definición:** Enlaza múltiples checkboxes con un array.

```Vue
  <p>Seleccionados: {{ selectedValues }}</p>

<script setup>

```

(Los valores de los checkboxes marcados se agregan al array selectedValues.)

## J - Value Bindings: Checkbox

**Definición:** Se pueden usar true-value y false-value para definir los valores que se asignan a la variable cuando el checkbox está marcado o desmarcado.

**Ejemplo:**

```Vue
<template>
  <input type="checkbox" v-model="toggle" true-value="yes" false-value="no" />
  <p>Valor: {{ toggle }}</p>
</template>

<script setup>
import { ref } from 'vue';
const toggle = ref('no');
</script>
```

(La variable toggle será "yes" si el checkbox está marcado, y "no" si está desmarcado.)

## K - Value Bindings: Radio

**Definición:** Se puede usar :value para enlazar el valor de un radio button a una propiedad dinámica.

**Ejemplo:**

```Vue
<template>
  <input type="radio" v-model="pick" :value="first" />
  <input type="radio" v-model="pick" :value="second" />
  <p>Seleccionado: {{ pick }}</p>
</template>

<script setup>
import { ref } from 'vue';
const first = ref('A');
const second = ref('B');
const pick = ref('');
</script>
```

(La variable pick se sincroniza con el valor de first o second según el radio button seleccionado.)

## L - Value Bindings: Select Options

**Definición:** Se puede usar :value para enlazar el valor de una opción de un `<select>` a una propiedad dinámica, incluso a objetos.

**Ejemplo:**

```Vue
<template>
  <select v-model="selected">
    <option :value="{ number: 123 }">123</option>
  </select>
  <p>Seleccionado: {{ selected }}</p>
</template>

<script setup>
import { ref } from 'vue';
const selected = ref(null);
</script>
```

(La variable selected se sincroniza con el objeto { number: 123 } cuando se selecciona la opción.)

## M - Modifiers: .lazy

**Definición:** Sincroniza el valor después del evento change en lugar del evento input.

**Ejemplo:**

```Vue
<template>
  <input v-model.lazy="message" />
  <p>Mensaje: {{ message }}</p>
</template>

<script setup>
import { ref } from 'vue';
const message = ref('');
</script>
```

(La variable message se actualiza solo cuando el campo pierde el foco.)

## N - Modifiers: .number

**Definición:** Convierte automáticamente el valor del input a un número.

**Ejemplo:**

```Vue
<template>
  <input v-model.number="age" />
  <p>Edad: {{ age }}</p>
</template>

<script setup>
import { ref } from 'vue';
const age = ref(0);
</script>
```

(La variable age se actualiza con un número.)

```Vue
</script>
```

(La variable message se actualiza sin espacios en blanco al inicio y al final.)
