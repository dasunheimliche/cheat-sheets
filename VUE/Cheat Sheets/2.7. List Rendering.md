## A - v-for

**Definición:** Directiva para renderizar una lista de elementos basada en un array. Usa la sintaxis item in items o item of items.

**Ejemplo:**

```Vue
<template>
  <ul>
    <li v-for="item in items">{{ item.message }}</li>
  </ul>
</template>

<script setup>
import { ref } from 'vue';
const items = ref([{ message: 'Foo' }, { message: 'Bar' }]);
</script>
```

(Renderiza una lista de li con los mensajes del array items.)

## B - v-for with Index

**Definición:** Se puede obtener el índice del elemento actual usando la sintaxis (item, index) in items.

**Ejemplo:**

```Vue
<template>
  <ul>
    <li v-for="(item, index) in items">
      {{ index }} - {{ item.message }}
    </li>
  </ul>
</template>

<script setup>
import { ref } from 'vue';
const items = ref([{ message: 'Foo' }, { message: 'Bar' }]);
</script>
```

(Muestra el índice y el mensaje de cada elemento.)

## C - Destructuring in v-for

**Definición:** Se puede usar la desestructuración para acceder directamente a las propiedades de los objetos en el array.

**Ejemplo:**

```Vue
<template>
  <ul>
    <li v-for="{ message } in items">{{ message }}</li>
  </ul>
</template>

<script setup>
import { ref } from 'vue';
const items = ref([{ message: 'Foo' }, { message: 'Bar' }]);
</script>
```

(Accede directamente a la propiedad message de cada objeto.)

## D - Nested v-for

**Definición:** Se pueden anidar directivas v-for para iterar sobre arrays dentro de arrays.

**Ejemplo:**

```Vue
<template>
  <ul>
    <li v-for="item in items">
      <span v-for="childItem in item.children">
        {{ item.message }} - {{ childItem }}
      </span>
    </li>
  </ul>
</template>

<script setup>
import { ref } from 'vue';
const items = ref([
  { message: 'A', children: ['a1', 'a2'] },
  { message: 'B', children: ['b1', 'b2'] }
]);
</script>
```

(Itera sobre los arrays items y children.)

## E - v-for with Objects

**Definición:** Se puede usar v-for para iterar sobre las propiedades de un objeto.

**Ejemplo:**

```Vue
<template>
  <ul>
    <li v-for="(value, key) in myObject">
      {{ key }}: {{ value }}
    </li>
  </ul>
</template>

<script setup>
import { reactive } from 'vue';
const myObject = reactive({
  title: 'Título',
  author: 'Autor'
});
</script>
```

(Muestra las claves y valores del objeto myObject.)

## F - v-for with a Range

**Definición:** Se puede usar v-for con un número entero para repetir un template un número específico de veces.

**Ejemplo:**

```Vue
<template>
  <span>
    <span v-for="n in 5">{{ n }}</span>
  </span>
</template>
```

(Repite el span 5 veces, mostrando los números del 1 al 5.)

## G - v-for on `<template>`

**Definición:** Se puede usar v-for en un elemento `<template>` para renderizar múltiples elementos. El `<template>` no se renderiza en el DOM.

**Ejemplo:**

```Vue
<template>
  <ul>
    <template v-for="item in items">
      <li>{{ item.msg }}</li>
      <li class="divider"></li>
    </template>
  </ul>
</template>

<script setup>
import { ref } from 'vue';
const items = ref([{ msg: 'A' }, { msg: 'B' }]);
</script>
```

(Renderiza una lista de li con un separador entre cada uno.)

## H - v-for with v-if

**Definición:** Cuando se usan v-if y v-for en el mismo elemento, v-if se evalúa primero. No se recomienda usar ambas directivas en el mismo elemento.

**Ejemplo:**

```Vue
<!-- No recomendado -->
<li v-for="item in items" v-if="item.isVisible">
  {{ item.name }}
</li>
```

(Es mejor usar una propiedad computada para filtrar los elementos antes de iterar.)

## I - Maintaining State with key

**Definición:** Se debe usar el atributo :key con v-for para que Vue pueda rastrear la identidad de cada elemento y reutilizar los nodos del DOM.

**Ejemplo:**

```Vue
<template>
  <ul>
    <li v-for="item in items" :key="item.id">
      {{ item.name }}
    </li>
  </ul>
</template>

<script setup>
import { ref } from 'vue';
const items = ref([
  { id: 1, name: 'A' },
  { id: 2, name: 'B' }
]);
</script>
```

(El atributo :key ayuda a Vue a identificar los elementos al actualizar la lista.)

## J - v-for with Components

**Definición:** Se puede usar v-for directamente en un componente, pero se deben pasar los datos a través de props.

**Ejemplo:**

```Vue
<template>
  <MyComponent
    v-for="(item, index) in items"
    :item="item"
    :index="index"
    :key="item.id"
  />
</template>

<script setup>
import { ref } from 'vue';
import MyComponent from './MyComponent.vue';
const items = ref([
  { id: 1, name: 'A' },
  { id: 2, name: 'B' }
]);
</script>
```

(Pasa los datos de cada elemento a través de props al componente MyComponent.)

## K - Array Mutation Methods

**Definición:** Métodos que mutan un array reactivo y activan actualizaciones en la interfaz de usuario: push(), pop(), shift(), unshift(), splice(), sort(), reverse().

**Ejemplo:**

```Js
items.value.push({ message: 'C' });
```

(Agrega un nuevo elemento al array y actualiza la interfaz de usuario.)

## L - Replacing an Array

**Definición:** Métodos que no mutan el array original, sino que retornan uno nuevo: filter(), concat(), slice(). Se debe reemplazar el array original con el nuevo.

**Ejemplo:**

```Js
items.value = items.value.filter((item) => item.message.includes('Foo'));
```

(Filtra el array y actualiza la interfaz de usuario.)

## M - Displaying Filtered/Sorted Results

**Definición:** Se pueden usar propiedades computadas o métodos para mostrar versiones filtradas o ordenadas de un array sin mutar el original.

**Ejemplo:**

```Vue
<template>
  <ul>
    <li v-for="n in evenNumbers">{{ n }}</li>
  </ul>
</template>

<script setup>
import { ref, computed } from 'vue';
const numbers = ref([1, 2, 3, 4, 5]);
const evenNumbers = computed(() => numbers.value.filter((n) => n % 2 === 0));
</script>
```

(Muestra solo los números pares del array numbers.)
