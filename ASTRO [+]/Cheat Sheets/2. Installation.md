### A - Prerrequisitos para Instalar Astro 🔴

#### 1. **Introducción:**

Antes de empezar a construir tu increíble sitio web, necesitas asegurarte de que tu computadora tenga las herramientas básicas instaladas, como si fueras a cocinar y necesitaras tener la cocina lista.

#### 2. **Ejemplo:**

Imagina que tu computadora es un taller. Para trabajar con Astro, necesitas:

- **Un motor (Node.js):** La versión `v18.20.8`, `v20.3.0`, `v22.0.0` o una superior. ¡Ojo! Las versiones 19 y 21 no son compatibles, es un detalle técnico, pero crucial.
- **Tu caja de herramientas (Editor de Texto):** Te recomiendo con todo mi ser [VS Code](https://code.visualstudio.com/) junto con la [extensión oficial de Astro](https://marketplace.visualstudio.com/items?itemName=astro-build.astro-vscode). Esto es como tener una caja de herramientas que ya sabe qué tornillos vas a usar.
- **El panel de control (Terminal):** Es esa ventana de texto donde escribes comandos. Astro se maneja desde ahí, ¡es tu centro de operaciones!

#### 3. **Desarrollo:**

Estos tres elementos no son opcionales; son la base sobre la que se construye todo.

- **Node.js:** Es el entorno que permite que Astro (que está escrito en JavaScript) funcione en tu computadora, fuera del navegador. Sin él, Astro no puede ni arrancar.
- **Editor de Texto con Extensión:** Podrías usar un bloc de notas, pero usar VS Code con la extensión de Astro te dará superpoderes: autocompletado, resaltado de sintaxis específica de Astro, y detección de errores al instante. ¡Te ahorrará horas de frustración!
- **Terminal:** Al principio puede intimidar, pero es la forma más directa y potente de comunicarte con herramientas de desarrollo como Astro. Comandos como `npm`, `pnpm` o `yarn` se ejecutan aquí.

🔴 **Fundamental**: No puedes empezar sin esto. Es como intentar construir un coche sin tener un motor, herramientas o un lugar donde ensamblarlo. Comprueba que tienes todo esto antes de escribir una sola línea de código.

### B - Instalación Guiada con el Asistente (`create astro`) 🔴

#### 1. **Introducción:**

Esta es la forma mágica y recomendada de empezar un proyecto con Astro, donde un asistente te toma de la mano y te hace preguntas para configurar todo por ti.

#### 2. **Ejemplo:**

Abre tu terminal (ese panel de control del que hablamos) y escribe uno de estos comandos, dependiendo de tu gestor de paquetes preferido (si no sabes cuál tienes, `npm` es el más común y viene con Node.js).

```bash
# Si usas npm (lo más probable)
npm create astro@latest

# Si usas pnpm
pnpm create astro@latest

# Si usas yarn
yarn create astro
```

**Explicación del ejemplo:**
Este comando le dice a tu gestor de paquetes (`npm`, `pnpm` o `yarn`) que quieres "crear" un nuevo proyecto de "astro" usando la "última" (`@latest`) versión disponible. Inmediatamente después, se lanzará un asistente que te preguntará el nombre del proyecto, si quieres usar una plantilla, instalar dependencias, etc. ¡Solo tienes que responder!

#### 3. **Desarrollo:**

El asistente `create astro` es tu mejor amigo al empezar. No necesitas crear una carpeta vacía antes; si no existe, el asistente la creará por ti. Te preguntará cosas como:

- ¿Dónde quieres crear tu proyecto?
- ¿Quieres empezar con una plantilla (blog, portafolio, etc.) o un proyecto vacío?
- ¿Quieres instalar las dependencias ahora? (Di que sí, te ahorra un paso).
- ¿Quieres usar TypeScript? (Si estás empezando, puedes elegir la opción más permisiva o "base").
- ¿Quieres inicializar un repositorio de Git? (Si usas Git, es muy útil).

Una vez que el asistente termina, solo tienes que entrar en la nueva carpeta (`cd mi-proyecto-astro`) y, si no instalaste las dependencias, ejecutar `npm install`. ¡Listo!

🔴 **Fundamental**: Este es el método de instalación principal y el que deberías usar el 99% de las veces. Es rápido, seguro y te configura todo siguiendo las mejores prácticas desde el primer segundo.

### C - Usar una Plantilla o Tema (`--template`) 🟡

#### 1. **Introducción:**

En lugar de empezar desde cero, puedes usar el asistente de instalación para crear un proyecto basado en una plantilla o un tema ya existente.

#### 2. **Ejemplo:**

Supongamos que quieres crear un blog y te gustó una plantilla de la comunidad Astro. Usarías el comando de creación (`create astro`) pero añadiendo la bandera `--template`.

```bash
# Usando una plantilla oficial de Astro (ej: un blog)
npm create astro@latest -- --template blog

# Usando un tema de un repositorio de GitHub
# Sintaxis: npm create astro@latest -- --template <usuario-github>/<nombre-repo>
npm create astro@latest -- --template withastro/astro-theme-blog
```

**Explicación del ejemplo:**
La bandera `--template` le dice al asistente: "Oye, no me hagas las preguntas de siempre. Quiero que clones este proyecto específico y lo uses como base para el mío". Los dos guiones (`--`) antes de la bandera son importantes cuando usas `npm` para asegurarte de que el argumento `--template` se le pasa al script de Astro y no al propio `npm`.

#### 3. **Desarrollo:**

Esto es increíblemente útil para acelerar tu desarrollo. ¿Para qué reinventar la rueda? Puedes encontrar cientos de plantillas en la [galería de temas de Astro](https://astro.build/themes/) para blogs, portafolios, sitios de documentación, etc. Simplemente encuentras el que te gusta, copias su nombre de GitHub y lo usas en el comando. Incluso puedes especificar una rama concreta del repositorio si lo necesitas (`usuario/repo#rama-especifica`).

🟡 **Importante**: Aunque no es estrictamente fundamental para _aprender_ Astro, es una herramienta extremadamente práctica y común en el mundo real para empezar proyectos rápidamente. Conocerla te ahorrará muchísimo tiempo.

### D - Añadir Integraciones Durante la Instalación (`--add`) 🟡

#### 1. **Introducción:**

Puedes decirle al asistente de instalación que configure automáticamente herramientas adicionales, como React, Tailwind CSS o Svelte, desde el primer momento.

#### 2. **Ejemplo:**

Imagina que ya sabes que tu proyecto va a necesitar componentes de React y la optimización de scripts de Partytown. Puedes instalarlos juntos con el proyecto.

```bash
# Instalar Astro junto con React y Partytown
npm create astro@latest -- --add react --add partytown
```

**Explicación del ejemplo:**
La bandera `--add` le indica al asistente que, además de crear el proyecto base de Astro, debe ejecutar el comando `astro add` para cada integración que le pases. En este caso, configurará todo lo necesario para que puedas usar componentes de React y Partytown sin pasos adicionales.

#### 3. **Desarrollo:**

Las "integraciones" son como plugins que le dan superpoderes a Astro. Te permiten usar herramientas de otros ecosistemas (como frameworks de UI) o añadir funcionalidades (como sitemaps automáticos o optimización de imágenes). Usar la bandera `--add` en la instalación inicial es simplemente un atajo para no tener que instalarlas una por una después con el comando `npx astro add react`.

🟡 **Importante**: A medida que tus proyectos crezcan, casi siempre usarás alguna integración. Saber que puedes añadirlas desde el inicio es un truco muy útil que te agiliza el flujo de trabajo.

### E - Instalación Manual (El Camino del Guerrero) 🔵

#### 1. **Introducción:**

Esta es la forma de instalar Astro pieza por pieza, ideal si quieres entender exactamente qué hace cada archivo de configuración y tener control total sobre el proceso.

#### 2. **Ejemplo:**

No hay un solo comando, sino una secuencia de pasos. Aquí está el resumen de la "receta":

1.  `mkdir mi-proyecto && cd mi-proyecto` (Crea y entra a la carpeta).
2.  `npm init -y` (Crea el archivo `package.json`).
3.  `npm install astro` (Instala Astro como dependencia).
4.  Añade los scripts (`dev`, `build`, `preview`) a tu `package.json`.
5.  Crea tu primera página en `src/pages/index.astro`.
6.  (Opcional pero recomendado) Crea los archivos `astro.config.mjs` y `tsconfig.json`.

#### 3. **Desarrollo:**

Este camino es más largo y propenso a errores si eres principiante, pero es pedagógicamente invaluable. Te obliga a entender el propósito de cada parte:

- **`package.json`**: El DNI de tu proyecto. Guarda el nombre, la versión y, lo más importante, la lista de dependencias (como `astro`) y los scripts para ejecutarlo.
- **`npm install astro`**: Este comando descarga el código de Astro y lo guarda en una carpeta llamada `node_modules`. Es crucial que lo instales localmente en tu proyecto, no globalmente en tu sistema.
- **`src/pages/`**: Astro usa un sistema de enrutamiento basado en archivos. Cualquier archivo `.astro`, `.md` o `.html` que pongas en esta carpeta se convertirá en una página de tu sitio web.
- **`astro.config.mjs`**: El cerebro de tu proyecto. Aquí configuras integraciones, rutas de salida y otros comportamientos de Astro.
- **`tsconfig.json`**: Le da contexto a tu editor de código (como VS Code) sobre tu proyecto, permitiendo un mejor autocompletado e inteligencia, incluso si no escribes una sola línea de TypeScript.

🔵 **Específico**: Este método no es para el día a día. Es para situaciones donde necesitas un control granular o, como en nuestro caso, para aprender cómo funciona Astro por dentro. La mayoría de las veces, usarás el [asistente de instalación](#b---instalación-guiada-con-el-asistente-create-astro-).
