## A - Páginas en Astro

**Definición:** Las páginas son archivos dentro del directorio src/pages/ de un proyecto Astro que definen las rutas, la carga de datos y la estructura general de cada página web en tu sitio. Son los puntos de entrada para que los usuarios accedan a tu contenido.

**Ejemplo:**

```astro
src/pages/about.astro
---
// Frontmatter para la carga de datos o importaciones de componentes
---
<html lang="en">
  <head>
    <title>Sobre Nosotros</title>
  </head>
  <body>
    <h1>Sobre Nuestra Empresa</h1>
    <p>Conoce más sobre lo que hacemos.</p>
  </body>
</html>
```

(Este archivo about.astro creará una página accesible en tudominio.com/about)

## B - Tipos de Archivos de Página Soportados

**Definición:** Astro soporta varios tipos de archivos dentro del directorio src/pages/, cada uno con su propio propósito y características. Estos incluyen .astro, .md, .mdx, .html y .js/.ts (para endpoints).

**Ejemplo:**

**.astro:** src/pages/index.astro (Componente Astro para páginas completas)

**.md:** src/pages/blog/my-first-post.md (Contenido Markdown para publicaciones de blog)

**.mdx:** src/pages/docs/installation.mdx (Markdown con componentes JSX incrustados)

**.html:** src/pages/legacy-page.html (Páginas HTML directas)

**.js/.ts:** src/pages/api/users.js (Endpoints de API)
(Cada tipo de archivo se trata como una página o endpoint, según su extensión)

## C - Enrutamiento Basado en Archivos

**Definición:** Astro utiliza enrutamiento basado en archivos, donde la ruta del archivo dentro de src/pages/ determina directamente la URL de la página. Por ejemplo, src/pages/blog/post.astro se convierte en tudominio.com/blog/post.

**Ejemplo:**

src/pages/index.astro -> tudominio.com/

src/pages/products/details.astro -> tudominio.com/products/details

src/pages/contact/index.astro -> tudominio.com/contact
(La estructura de carpetas dentro de src/pages refleja la estructura de URL de tu sitio)

## D - Enlaces Entre Páginas

**Definición:** Utiliza elementos HTML estándar con rutas URL relativas a tu dominio raíz para enlazar entre páginas en tu sitio Astro. Evita usar rutas de archivos relativas.

**Ejemplo:**

```html
<a href="/blog/my-post/">Lee mi publicación de blog</a>
```

(Este enlace llevará al usuario a tudominio.com/blog/my-post/, independientemente de la ubicación de la página actual)

## E - Páginas Astro (.astro)

**Definición:** Las páginas Astro son archivos .astro dentro de src/pages/ que utilizan componentes y sintaxis de Astro para crear documentos HTML completos. Pueden importar componentes, usar frontmatter y renderizar contenido dinámico.

**Ejemplo:**

```astro
---
import Layout from '../layouts/Layout.astro';
const pageTitle = "Mi Página Asombrosa";
---
<Layout title={pageTitle}>
  <h1>¡Bienvenido a mi página!</h1>
  <p>Este es algo de contenido.</p>
</Layout>
```

(Este ejemplo muestra una página Astro básica usando un componente de diseño y una variable)

## F - Páginas Markdown/MDX (.md/.mdx)

**Definición:** Astro trata los archivos .md (Markdown) y .mdx (Markdown con JSX) en src/pages/ como páginas. Pueden usar frontmatter para especificar diseños y otros metadatos.

**Ejemplo:**

```markdown
---
layout: "../layouts/BlogPostLayout.astro"
title: "Mi Primer Post de Blog"
date: "2024-01-26"
---

# Mi Primer Post de Blog

Este es el contenido de mi post de blog.
```

(Este archivo Markdown se renderizará como una página, envuelto en el diseño especificado)

## G - Páginas HTML (.html)

**Definición:** Los archivos .html en src/pages/ se utilizan directamente como páginas. Sin embargo, no soportan todas las características de Astro como frontmatter o importaciones de componentes.

**Ejemplo:**

```html
<!DOCTYPE html>
<html lang="en">
  <head>
    <title>Página Legado</title>
  </head>
  <body>
    <h1>Esta es una página HTML legado.</h1>
  </body>
</html>
```

(Este archivo HTML se servirá tal cual, sin ningún procesamiento de Astro)

## H - Página de Error 404 Personalizada

**Definición:** Crea un archivo 404.astro o 404.md en src/pages/ para definir una página de error 404 personalizada que se mostrará cuando un usuario intente acceder a una página inexistente.

**Ejemplo:**

```astro
// src/pages/404.astro
---
---
<html lang="en">
  <head>
    <title>Página No Encontrada</title>
  </head>
  <body>
    <h1>404 - Página No Encontrada</h1>
    <p>Lo sentimos, la página que buscabas no pudo ser encontrada.</p>
  </body>
</html>
```

(Esta página se mostrará cuando un usuario navegue a una URL inexistente)

## I - Página de Error 500 Personalizada

**Definición:** Crea un archivo 500.astro en src/pages/ para definir una página de error 500 personalizada que se mostrará cuando ocurra un error durante la renderización del lado del servidor. Esta página recibe una propiedad error con detalles sobre el error.

**Ejemplo:**

```astro
// src/pages/500.astro
---
interface Props {
    error: unknown
}

const { error } = Astro.props
---
<html lang="en">
  <head>
    <title>Error del Servidor</title>
  </head>
  <body>
    <h1>500 - Error del Servidor</h1>
    <p>Ocurrió un error: {error instanceof Error ? error.message : 'Error desconocido'}</p>
  </body>
</html>
```

(Esta página se mostrará cuando ocurra un error del lado del servidor, mostrando el mensaje de error si está disponible)

## J - Fragmentos de Página

**Definición:** Los fragmentos de página son archivos .astro o .mdx en src/pages/ que están marcados con export const partial = true;. No están destinados a ser páginas completas, sino fragmentos de HTML que pueden cargarse dinámicamente en una página usando una biblioteca de front-end como htmx. No incluyen la declaración o el contenido.

**Ejemplo:**

```astro
// src/pages/partials/my-partial.astro
---
export const partial = true;
---
<p>Este es un fragmento HTML parcial.</p>
```

(Este fragmento puede cargarse en una página usando htmx o bibliotecas similares)

## K - Uso de Fragmentos con una Biblioteca (htmx)

**Definición:** Los fragmentos a menudo se usan con bibliotecas como htmx para actualizar dinámicamente partes de una página sin recargar toda la página. La biblioteca envía una solicitud a la URL del fragmento, y la respuesta se usa para actualizar un elemento específico en la página.

**Ejemplo:**

```astro
// src/pages/index.astro
<html>
  <head>
    <title>Mi página</title>
    <script src="https://unpkg.com/htmx.org@1.9.6"></script>
  </head>
</html>
<section>
  <div id="target-div">Contenido Inicial</div>
  <button hx-get="/partials/my-partial/" hx-target="#target-div" hx-swap="innerHTML">
    Cargar Fragmento
  </button>
</section>

// src/pages/partials/my-partial.astro
---
export const partial = true;
---
<p>¡Contenido parcial cargado!</p>
```

(Al hacer clic en el botón, se cargará el contenido de my-partial.astro en el elemento target-div)
