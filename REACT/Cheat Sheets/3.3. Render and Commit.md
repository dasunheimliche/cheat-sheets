## C - Paso 3: React Commits Changes to the DOM (React Aplica los Cambios al DOM) üçΩÔ∏è

#### 1. **Definici√≥n:**

Despu√©s de que React renderiza tus componentes y calcula qu√© debe mostrarse, llega el momento de **aplicar esos cambios al DOM** (el Document Object Model, que es la representaci√≥n de tu p√°gina web en el navegador). Este paso se llama **"commit"**.

#### 2. **Analog√≠a del Restaurante:**

#### 3. **Detalles Clave:**

- **Actualizaci√≥n M√≠nima:** React **solo cambia los nodos del DOM si hay diferencias** entre renders. Si algo no ha cambiado, React **no lo toca**. Esto es muy importante para el rendimiento.
- **Ejemplo Input:** Si tienes un `<input>` en tu componente y este se re-renderiza (por ejemplo, porque un componente padre actualiza sus props), **el texto que hayas escrito en el `<input>` no desaparece**.
